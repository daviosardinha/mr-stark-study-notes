Phase 2: Authenticated Enumeration

The Lab Scenario (DefCorp)

Understanding your starting position is vital. In this guide, you are not usually starting as a random hacker on the internet trying to guess passwords. You are in an Assumed Breach scenario.

The Setup:

  • The Target: DefCorp, a defense contractor with high security requirements.
  • The Environment: A Hybrid setup. This means they have an On-Premise Active Directory that syncs users to the Azure Cloud. This creates a massive attack surface.
  • Your Role: "External Contractor." You have been given legitimate, low-privilege credentials (email and password) to perform maintenance work.

Authentication Bypass and Token Extraction

This section is the bridge between Phase 1 (Outside) and Phase 2 (Inside). Before we can perform authenticated enumeration, we must solve a critical problem.

We have the credentials for test@defcorphq.onmicrosoft.com but logging directly into the administrative portal blocks us because of Mandatory MFA Enforcement.

In an assumed breach scenario, possessing a username and password is no longer enough. Since late 2024, Microsoft has enforced Mandatory Multi-Factor Authentication (MFA) for all users attempting to sign into Azure Management portals.

However, this security control has a specific weakness we can exploit. The "Mandatory MFA" policy targets specific administrative Cloud Applications like the "Azure Portal" or "Intune Admin Center." It often ignores standard user productivity portals.

Device Code Flow (The Bypass)

The device code flow is an OAuth 2.0 authentication method designed for devices with limited input capabilities. It allows authentication without requiring the user to enter credentials directly on the device.

az login --tenant defcorphq.onmicrosoft.com

This will provide a URL and code. The user authenticates on another device (their phone or a clean browser), and the tool receives the token.

Enumeration - Mg Module (Microsoft Graph PowerShell)

This module allows you to interact directly with the Microsoft Graph API, which is the "control plane" for Entra ID (Identity).

Connection & Session (Pass-the-Token)

In a typical Red Team scenario, you often obtain an access token through phishing (Evilginx), device code flow, or by extracting it from a compromised machine's browser memory or token cache.

Connect using interactive login:

Connect-MgGraph

Enumerating Users

Get-MgUser | Select-Object DisplayName, UserPrincipalName, JobTitle

Enumerating Groups

Get-MgGroup | Select-Object DisplayName, Description

Enumerating Group Members

Get-MgGroupMember -GroupId "group-id" | Select-Object DisplayName, UserPrincipalName

Enumerating Service Principals

Get-MgServicePrincipal | Select-Object DisplayName, AppId

Enumerating Devices

Get-MgDevice | Select-Object DisplayName, OperatingSystem, TrustType

Enumerating Subscriptions

Get-MgSubscription

2. Enumeration - Az PowerShell (Azure Resource Manager)

This is the standard administrative tool for managing infrastructure. While Mg Module focused on Identity, this module focuses on Resources (VMs, Storage, Key Vaults).

Connection (Identity & Resource Hybrid)

In advanced attacks, you often possess separate tokens for Graph (Identity) and ARM (Infrastructure).

Connect-AzAccount

Infrastructure Mapping

This is your primary reconnaissance command. It returns an inventory of everything you have access to.

Get-AzResource | Select-Object Name, ResourceGroupName, Type

Enumerating Subscriptions

Get-AzSubscription

Enumerating Resource Groups

Get-AzResourceGroup

Enumerating Virtual Machines

Get-AzVM

Enumerating Storage Accounts

Get-AzStorageAccount

Enumerating Key Vaults

Key Vaults store secrets, certificates, and encryption keys. They are high-value targets.

az keyvault list --output table

Listing Key Vault Secrets

If you have permissions, enumerate secrets stored in Key Vaults.

az keyvault secret list --vault-name "VaultName" --output table

Enumerating App Services

az webapp list --output table

Listing Function Apps

az functionapp list --output table

Enumerating Role Assignments

Understanding who has what permissions is crucial for privilege escalation assessment.

az role assignment list --output table

Listing Custom Roles

az role definition list --output table

Enumerating Network Interfaces

Network interfaces reveal IP configurations and can expose internal networks.

az network nic list --output table

Listing Private Endpoints

Private endpoints enable private access to Azure services without internet exposure.

az network private-endpoint list --output table

JMESPath Query Examples

Advanced filtering capabilities of Azure CLI:

# Get only VMs in a specific resource group
az vm list --query "[?resourceGroup=='RGName']" --output table

# Get storage accounts with specific tags
az storage account list --query "[?tags.environment=='prod']" --output table

# List resources by type
az resource list --query "[?type=='Microsoft.Compute/virtualMachines']" --output table

Enumeration - Azure CLI (az cli)

Priority: Rapid Assessment
Role: Cross-platform Querying & JSON Filtering
Context: Developers use this tool extensively. Its "JMESPath" query language allows you to filter huge datasets much faster than PowerShell loops.

Why Azure CLI is Essential for Red Teams

  • Speed: JMESPath queries can filter data server-side, reducing local processing
  • Cross-Platform: Works on Windows, macOS, and Linux
  • Automation: Easy to script for automated enumeration
  • JSON Output: Structured data perfect for further processing

Authentication

This initiates an interactive login in the browser.

az login

Listing Subscriptions

Subscriptions are the primary billing and access boundary in Azure. Enumerating all accessible subscriptions is the first step in Azure resource assessment.

az account list --output table

Setting Subscription

After identifying subscriptions, set the context to enumerate resources within each.

az account set --subscription "subscription-name"

Enumerating Resource Groups

Resource groups are logical containers for Azure resources.

az group list --output table

Listing Resources

az resource list --output table

Listing VMs

az vm list --output table

Listing Storage Accounts

az storage account list --output table

Getting Storage Account Keys

Storage account keys provide access to blob containers, files, queues, and tables.

az storage account keys list --resource-group "RGName" --account-name "StorageName"

Listing Key Vaults

az keyvault list --output table

Listing Web Apps

az webapp list --output table

Listing Role Assignments

az role assignment list --output table

Using JMESPath Queries

Filter results using JMESPath queries:

az vm list --query "[?location=='eastus']" --output table
az resource list --query "[?type=='Microsoft.Compute/virtualMachines']" --output table
