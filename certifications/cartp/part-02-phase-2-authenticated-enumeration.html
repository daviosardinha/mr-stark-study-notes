Phase 2: Authenticated Enumeration

The Lab Scenario (DefCorp)

Understanding your starting position is vital. In this guide, you are not usually starting as a random hacker on the internet trying to guess passwords. You are in an Assumed Breach scenario.

The Setup:

  • The Target: DefCorp, a defense contractor with high security requirements.
  • The Environment: A Hybrid setup. This means they have an On-Premise Active Directory that syncs users to the Azure Cloud. This creates a massive attack surface.
  • Your Role: "External Contractor." You have been given legitimate, low-privilege credentials (email and password) to perform maintenance work.

Authentication Bypass and Token Extraction

This section is the bridge between Phase 1 (Outside) and Phase 2 (Inside). Before we can perform authenticated enumeration, we must solve a critical problem.

We have the credentials for test@defcorphq.onmicrosoft.com but logging directly into the administrative portal blocks us because of Mandatory MFA Enforcement.

In an assumed breach scenario, possessing a username and password is no longer enough. Since late 2024, Microsoft has enforced Mandatory Multi-Factor Authentication (MFA) for all users attempting to sign into Azure Management portals.

However, this security control has a specific weakness we can exploit. The "Mandatory MFA" policy targets specific administrative Cloud Applications like the "Azure Portal" or "Intune Admin Center." It often ignores standard user productivity portals.

Device Code Flow (The Bypass)

The device code flow is an OAuth 2.0 authentication method designed for devices with limited input capabilities. It allows authentication without requiring the user to enter credentials directly on the device.

az login --tenant defcorphq.onmicrosoft.com

This will provide a URL and code. The user authenticates on another device (their phone or a clean browser), and the tool receives the token.

Enumeration - Mg Module (Microsoft Graph PowerShell)

This module allows you to interact directly with the Microsoft Graph API, which is the "control plane" for Entra ID (Identity).

Connection & Session (Pass-the-Token)

In a typical Red Team scenario, you often obtain an access token through phishing (Evilginx), device code flow, or by extracting it from a compromised machine's browser memory or token cache.

Connect using interactive login:

Connect-MgGraph

Enumerating Users

User enumeration is critical for understanding the identity landscape:

# List all users with key properties
Get-MgUser -All | Select-Object DisplayName, UserPrincipalName, JobTitle, Mail, UserType

# Get specific user details
Get-MgUser -UserId "user@domain.com" | Select-Object *

# Enumerate user licenses
(Get-MgUserLicenseDetail -UserId "user@domain.com").ServicePlans

# Get user's owned devices
Get-MgUserOwnedDevice -UserId "user@domain.com"

# Get user's registered devices
Get-MgUserRegisteredDevice -UserId "user@domain.com"

Enumerating Groups

Groups are the primary method for assigning permissions:

# List all security groups
Get-MgGroup -All | Where-Object { $_.GroupTypes -eq 'Security' } | Select-Object DisplayName, Description

# List Microsoft 365 groups
Get-MgGroup -All | Where-Object { $_.GroupTypes -eq 'Unified' } | Select-Object DisplayName, Description

# Get group members recursively
Get-MgGroupMemberTransitive -GroupId "group-id"

# Find user's group memberships
Get-MgUserMemberOf -UserId "user@domain.com"

Enumerating Group Members

Get-MgGroupMember -GroupId "group-id" | Select-Object DisplayName, UserPrincipalName

Enumerating Service Principals

Service principals are application identities - high-value targets for credential hunting:

# List all service principals
Get-MgServicePrincipal -All | Select-Object DisplayName, AppId, AccountEnabled

# Get service principal owners
Get-MgServicePrincipalOwner -ServicePrincipalId "app-id"

# Get app role assignments
Get-MgServicePrincipalAppRoleAssignment -ServicePrincipalId "app-id"

Enumerating Devices

Devices are increasingly important for conditional access and token theft:

# List all registered devices
Get-MgDevice -All | Select-Object DisplayName, OperatingSystem, OperatingSystemVersion, TrustType, DeviceId

# Get device owners
Get-MgDeviceOwner -DeviceId "device-id"

# Get device registered users
Get-MgDeviceRegisteredUser -DeviceId "device-id"

# Find compliant devices
Get-MgDevice -All | Where-Object { $_.IsCompliant -eq $true }

Enumerating Applications

Application registrations and enterprise applications reveal attack surfaces:

# List application registrations
Get-MgApplication -All | Select-Object DisplayName, AppId, IdentifierUris

# Get application owners
Get-MgApplicationOwner -ApplicationId "app-id"

# List enterprise applications (SSO integrations)
Get-MgServicePrincipal -All | Where-Object { $_.Tags -contains 'WindowsAzureActiveDirectoryIntegratedApp' }

# Get implicit grants (OAuth permissions granted)
Get-MgOAuth2PermissionGrant -All

Enumerating Directory Roles

Directory roles show who has administrative privileges:

# List all directory role templates
Get-MgDirectoryRoleTemplate

# Get activated directory roles
Get-MgDirectoryRole

# Get members of a directory role
Get-MgDirectoryRoleMember -DirectoryRoleId "role-id"

Enumerating Subscriptions

Subscriptions are the fundamental billing and access boundary in Azure:

# List all accessible subscriptions
Get-AzSubscription

# Get subscription details
Get-AzSubscription -SubscriptionId "sub-id" | Select-Object *

# Get subscription providers
Get-AzResourceProvider -SubscriptionId "sub-id"

# Get subscription tenants
Get-AzTenant

Enumerating Resource Groups

Resource groups organize resources and inherit permissions:

# List all resource groups
Get-AzResourceGroup | Select-Object ResourceGroupName, Location

# Get resources in a specific group
Get-AzResource -ResourceGroupName "RGName"

# Get resource providers
Get-AzResourceProvider -ListAvailable

Enumerating Role Assignments

Understanding who has what access is critical for privilege escalation:

# List all role assignments
Get-AzRoleAssignment

# Get role assignments for a specific user
Get-AzRoleAssignment -SignInName "user@domain.com"

# Get role definitions (available roles)
Get-AzRoleDefinition | Select-Object Name, Id, Description

# Check for eligible role assignments (PIM)
Get-AzRoleAssignment -ExpandPrincipalGroups | Where-Object { $_.RoleAssignmentScheduleId }

2. Enumeration - Az PowerShell (Azure Resource Manager)

This is the standard administrative tool for managing infrastructure. While Mg Module focused on Identity, this module focuses on Resources (VMs, Storage, Key Vaults).

Connection (Identity & Resource Hybrid)

In advanced attacks, you often possess separate tokens for Graph (Identity) and ARM (Infrastructure).

Connect-AzAccount

Infrastructure Mapping

This is your primary reconnaissance command. It returns an inventory of everything you have access to.

Get-AzResource | Select-Object Name, ResourceGroupName, Type

Enumerating Subscriptions

Get-AzSubscription

Enumerating Resource Groups

Get-AzResourceGroup

Enumerating Virtual Machines

Virtual machines are often high-value targets for code execution and lateral movement:

# List all VMs with details
Get-AzVM | Select-Object Name, ResourceGroupName, Location, PowerState

# Get VM extensions (may contain credentials)
Get-AzVMExtension -ResourceGroupName "RGName" -VMName "VMName"

# Get VM disk information
Get-AzDisk -ResourceGroupName "RGName"

# Get boot diagnostics
Get-AzVMDiagnosticsExtension -ResourceGroupName "RGName" -VMName "VMName"

Enumerating Storage Accounts

Storage accounts may contain sensitive data and can be used for data exfiltration:

# List storage accounts
Get-AzStorageAccount | Select-Object StorageAccountName, ResourceGroupName, Location

# Get storage account keys
Get-AzStorageAccountKey -ResourceGroupName "RGName" -Name "StorageName"

# List blob containers
Get-AzStorageContainer -Context (Get-AzStorageAccount -ResourceGroupName "RGName" -Name "StorageName").Context

# List file shares
Get-AzStorageShare -Context (Get-AzStorageAccount -ResourceGroupName "RGName" -Name "StorageName").Context

Enumerating Key Vaults

Key Vaults store secrets, certificates, and encryption keys. They are high-value targets.

# List key vaults
Get-AzKeyVault | Select-Object VaultName, ResourceGroupName, Location

# Get vault access policies
Get-AzKeyVault -VaultName "VaultName" | Select-Object AccessPolicies

# List secrets
Get-AzKeyVaultSecret -VaultName "VaultName"

# List certificates
Get-AzKeyVaultCertificate -VaultName "VaultName"

# List keys
Get-AzKeyVaultKey -VaultName "VaultName"

Enumerating App Services

Azure App Services host web applications and often have Managed Identities:

# List web apps
Get-AzWebApp | Select-Object Name, ResourceGroupName, State

# Get web app configuration
Get-AzWebApp -Name "AppName" -ResourceGroupName "RGName" | Select-Object *

# Get app settings (may contain secrets)
Get-AzWebAppApplicationSettings -Name "AppName" -ResourceGroupName "RGName"

# Check for managed identity
Get-AzWebApp -Name "AppName" -ResourceGroupName "RGName" | Select-Object Identity

Enumerating Function Apps

# List function apps
Get-AzFunctionApp | Select-Object Name, ResourceGroupName, State

# Get function app configuration
Get-AzFunctionApp -Name "FuncName" -ResourceGroupName "RGName"

Enumerating Network Interfaces

Network interfaces reveal IP configurations and can expose internal networks:

# List network interfaces
Get-AzNetworkInterface | Select-Object Name, ResourceGroupName, IpConfigurations

# Get IP configurations
Get-AzNetworkInterface -Name "NICName" | Select-Object -ExpandProperty IpConfigurations

Enumerating Private Endpoints

# List private endpoints
Get-AzPrivateEndpoint | Select-Object Name, ResourceGroupName, Subnet

# Get private endpoint connections
Get-AzPrivateEndpointConnection -ResourceGroupName "RGName"

Enumerating Azure SQL

SQL databases may contain sensitive data:

# List SQL servers
Get-AzSqlServer | Select-Object ServerName, ResourceGroupName, Location

# List databases
Get-AzSqlDatabase -ServerName "ServerName" -ResourceGroupName "RGName"

# Get SQL firewall rules
Get-AzSqlServerFirewallRule -ServerName "ServerName" -ResourceGroupName "RGName"

Enumeration - Azure CLI (az cli)

Listing Key Vault Secrets

If you have permissions, enumerate secrets stored in Key Vaults.

az keyvault secret list --vault-name "VaultName" --output table

Enumerating App Services

az webapp list --output table

Listing Function Apps

az functionapp list --output table

Enumerating Role Assignments

Understanding who has what permissions is crucial for privilege escalation assessment.

az role assignment list --output table

Listing Custom Roles

az role definition list --output table

Enumerating Network Interfaces

Network interfaces reveal IP configurations and can expose internal networks.

az network nic list --output table

Listing Private Endpoints

Private endpoints enable private access to Azure services without internet exposure.

az network private-endpoint list --output table

JMESPath Query Examples

Advanced filtering capabilities of Azure CLI:

# Get only VMs in a specific resource group
az vm list --query "[?resourceGroup=='RGName']" --output table

# Get storage accounts with specific tags
az storage account list --query "[?tags.environment=='prod']" --output table

# List resources by type
az resource list --query "[?type=='Microsoft.Compute/virtualMachines']" --output table

Enumeration - Azure CLI (az cli)

Priority: Rapid Assessment
Role: Cross-platform Querying & JSON Filtering
Context: Developers use this tool extensively. Its "JMESPath" query language allows you to filter huge datasets much faster than PowerShell loops.

Why Azure CLI is Essential for Red Teams

  • Speed: JMESPath queries can filter data server-side, reducing local processing
  • Cross-Platform: Works on Windows, macOS, and Linux
  • Automation: Easy to script for automated enumeration
  • JSON Output: Structured data perfect for further processing

Authentication

This initiates an interactive login in the browser.

az login

Subscriptions are the primary billing and access boundary in Azure. Enumerating all accessible subscriptions is the first step in Azure resource assessment.

az account list --output table

Setting Subscription

After identifying subscriptions, set the context to enumerate resources within each.

az account set --subscription "subscription-name"

Enumerating Resource Groups

Resource groups are logical containers for Azure resources.

az group list --output table

Listing Resources

az resource list --output table

Listing VMs

az vm list --output table

Listing Storage Accounts

az storage account list --output table

Getting Storage Account Keys

Storage account keys provide access to blob containers, files, queues, and tables.

az storage account keys list --resource-group "RGName" --account-name "StorageName"

Listing Key Vaults

az keyvault list --output table

Listing Web Apps

az webapp list --output table

Listing Role Assignments

az role assignment list --output table

Using JMESPath Queries

Filter results using JMESPath queries:

az vm list --query "[?location=='eastus']" --output table
az resource list --query "[?type=='Microsoft.Compute/virtualMachines']" --output table

Summary: Phase 2 Key Takeaways

Authenticated Enumeration is Critical:

  • Enumerate identity (users, groups, devices) using MgGraph
  • Enumerate resources (VMs, storage, key vaults) using Az PowerShell/CLI
  • Identify privilege escalation paths through role assignments

Key Enumeration Targets:

  • Users: Identify high-privilege accounts and service accounts
  • Groups: Find members of privileged security groups
  • Service Principals: Hunt for over-privileged application identities
  • Devices: Find registered devices for token theft opportunities
  • Role Assignments: Map permission paths to privileged roles

Enumeration Tools:

  • MgGraph PowerShell for Entra ID enumeration
  • Az PowerShell for Azure resource enumeration
  • Azure CLI for rapid assessment with JMESPath filtering
  • AzureHound + BloodHound for attack path analysis
